## 🎯 核心问题分析

### 问题1：非因果天气逻辑 ⚠️ 致命缺陷

**问题描述：**
- 模型声称具备气候适应性，但在预测未来交通时，仅依靠历史天气特征
- 完全没有将预测时间窗口内的实际天气（或天气预报）作为输入参数
- 违背了时间序列预测中的基本因果逻辑

**改进方案：**
```
修改数据加载机制 → 提取未来天气特征
修改模型架构 → 在解码器中显式注入未来天气
修改训练循环 → 处理新的数据格式
```

**需要修改的文件：** 4 个
- data_provider/data_loader.py
- models/CAST.py
- exp/exp_main.py
- run.py

**预期效果：** 模型性能提升 5-10%

**优先级：** 🔴 第一优先级（必须）

---

### 问题2：空间拓扑结构破坏 ⚠️ 严重缺陷

**问题描述：**
- 现有的全局聚合-分发（GAD）机制将整个交通网络的节点强行池化为一个全局向量
- 然后无差别地广播给所有节点
- 完全抹杀了交通路网原有的上下游物理连接和空间拓扑动态

**改进方案：**
```
实现双轨空间模块：
├─ 分支1：局部图卷积 → 保留物理拓扑
├─ 分支2：全局注意力 → 捕获宏观模式
└─ 融合：既保留约束，又捕获模式
```

**需要创建的文件：** 1 个
- layers/GCN.py

**预期效果：** 模型性能进一步提升 3-5%

**优先级：** 🟠 第二优先级（重要）

---

### 问题3：数据集设定模糊与前后不一致 ⚠️ 严谨性问题

**问题描述：**
- 论文声称只为 PeMS03 构建了真实天气数据集
- 但却报告了四个数据集在"有气象输入"下的对比结果
- 无天气版本的 PeMS04 在主表格中的结果与消融实验图中的数据存在明显矛盾

**改进方案：**
```
严格的实验分组：
├─ 第一组：标准数据集（无天气）
│  └─ ETTm1, ETTh1, PEMS04, PEMS07, PEMS08
├─ 第二组：气候融合数据集（有真实天气）
│  └─ PEMS03（核心验证集）
└─ 明确标记每个实验的数据集类型
```

**需要修改的文件：** 2 个
- run.py
- experiments_config.yaml（新增）

**预期效果：** 论文严谨性大幅提升

**优先级：** 🟡 第三优先级（关键）

---

### 问题4：消融实验与定性分析过于单薄 ⚠️ 支撑不足

**问题描述：**
- 消融实验只在一个数据集上进行
- 定性分析仅依赖于一个代表性案例
- 不足以支撑模型具备强鲁棒性的结论

**改进方案：**
```
扩展的消融实验：
├─ 在 PEMS03（有天气）上进行消融
├─ 多种天气条件的定性分析
│  ├─ 正常天气
│  ├─ 突发暴雨
│  └─ 持续极端高温
└─ 多个指标（MAE, RMSE, MAPE）
```

**需要创建的文件：** 2 个
- scripts/ablation_extended.sh
- analysis/qualitative_analysis.py

**预期效果：** 充分支撑"模型具备强鲁棒性"的结论

**优先级：** 🟢 第四优先级（补充）
